# Harbor Market 商品中心需求

## 状态与边界

- **Spec:** `harbor-product-management`
- **目标:** 为后台、H5 和微信小程序提供同一套商品数据源
- **渠道:** 中国大陆单商户；微信小程序优先，H5 继续保留
- **本阶段:** 商品、类目、SKU、图片、Excel 批量导入导出、管理员后台、私有 MinIO 集成
- **不包含:** 购物车、订单、支付、促销引擎、会员、远端部署
- **部署约束:** 本任务不得修改 `192.168.1.33`；由独立 review 任务审查、备份并部署

## 1. 管理员权限

1. 普通注册用户默认不是管理员。
2. 所有 `/api/v1/admin/**` 接口必须同时要求有效登录和管理员权限。
3. 未登录返回 `401`；已登录但非管理员返回 `403`。
4. 管理员提升必须通过显式 CLI 完成，禁止“第一个注册用户自动成为管理员”。
5. 当前用户响应必须包含 `is_admin`，但不得暴露密码哈希或会话内容。

## 2. 类目

1. 管理员可创建、编辑、排序、启停和删除未使用类目。
2. 类目包含稳定的 `code`、名称、说明、父类目、排序和启用状态。
3. 公开目录只显示启用类目。
4. 已被商品使用的类目不得直接删除。

## 3. 商品与 SKU

1. 商品以稳定 `product_code` 作为 Excel/API 幂等键。
2. 商品至少包含名称、副标题、类目、基础价、划线价、币种、单位、说明、标签、卖点、配料、过敏原、推荐标识、库存状态、库存、排序和状态。
3. 状态仅允许 `draft`、`published`、`archived`；公开目录只返回 `published`。
4. 金额以人民币“分”为数据库/API 整数单位，Excel 用人民币“元”表示。
5. 可选 SKU 以稳定 `sku_code` 唯一标识，包含名称、价格、划线价、库存、属性、默认标识、启用状态和排序。
6. 商品可保存规格/定制项 JSON，以覆盖杯型、冷热、糖度、冰量、加料等瑞幸式饮品选择，也可用于普通商品规格。
7. 上架商品必须具备启用类目、非负价格和且仅一张封面图。

## 4. 图片与私有对象存储

1. MinIO 只在 Compose 私有网络提供 S3 API，不得匿名公开 bucket，也不得把 9000 暴露给 LAN/公网。
2. Console 最多只绑定宿主机 `127.0.0.1:9001`。
3. 客户端经同源只读媒体 API 获取商品图片；所有写入由管理员 API 完成。
4. 支持 JPEG、PNG、WebP，单张最大 5 MiB，并验证真实图片格式和尺寸。
5. 每个商品允许：封面 1 张、轮播图 0–8 张、详情图 0–20 张。
6. 对象路径由服务端生成：直接上传使用 `products/{product_id}/{role}/{uuid}.{ext}`，Excel 暂存提升使用稳定编码目录 `products/catalog/{product_code}/{role}/{uuid}.{ext}`；不得信任用户提供的绝对路径或 `..`。
7. 数据库只保存对象 key 和 MIME、大小、宽高、替代文本、排序等元数据。
8. 删除数据库记录时应删除对应对象；失败时必须给出可恢复的明确错误。
9. 暂存图片保留 7 天，每个商品编码最多 100 个、全局最多 5,000 个；上传人、过期时间和清理状态必须持久化，后台 worker 自动回收。
10. 对象删除采用可重试 outbox；删除前必须再次确认没有 `product_images` 活跃引用。

## 5. API

管理员至少具备：

- 类目 CRUD
- 商品分页/搜索/过滤、创建、详情、更新、删除
- 商品图片上传、排序和删除
- Excel 模板下载、导入预检、原子导入和导出
- 导入结果与逐行错误报告
- 暂存图片上传/取消、最近导入任务和对象清理任务

公开目录至少具备：

- 启用类目列表
- 已上架商品分页/类目筛选/搜索
- 商品详情（SKU、规格和有序图片）
- 同源媒体读取

## 6. Excel 批量操作

1. 模板和导出文件采用 `.xlsx`，包含 `Products`、`SKUs`、`Images`、`Dictionary` 工作表。
2. `Products` 通过 `product_code` 新增或更新；`SKUs` 通过 `sku_code` 新增或更新。
3. `Images.object_key` 指向已上传到私有 MinIO 的对象路径，导入时必须检查对象存在。
4. 支持 `dry_run`：执行完整校验但不修改数据库。
5. 正式导入必须“先全量校验、后单事务写入”；任一错误时不得部分写入。
6. 错误包含工作表、行号、字段和中文可读说明。
7. 导出文件必须能够在不修改关键字段的情况下重新导入。
8. 模板应包含字段说明、示例、枚举和人民币元/分规则，冻结表头并提供筛选。
9. 正式导入接受 `X-Idempotency-Key`；同一 key、文件和模式重试返回原任务，不得重复写入，不同文件/模式复用 key 返回冲突。
10. 暂存图片提升必须先持久化目标清理意图；进程在复制后中断时，worker 能回收未关联的目标对象。

## 7. 后台 UI

1. 只有管理员能进入 `/admin/products`。
2. 商品列表支持搜索、状态/类目过滤、分页、新建和编辑入口。
3. 编辑页支持核心字段、SKU/规格 JSON、图片上传预览和删除。
4. Excel 页面支持模板下载、导出、dry-run、正式导入和错误显示。
5. 上传、保存和导入期间必须防止重复提交并显示进度/结果。
6. 页面应保留一次文件选择的导入幂等 key，并展示最近导入任务；暂存图片可主动取消。
7. 页面在桌面和移动宽度均不得出现关键控件溢出或遮挡。

## 8. 验收与质量门

1. Alembic 可从 `0001` 升级到商品版本并可回滚。
2. Ruff、后端测试、前端 lint/typecheck/test/build、Compose config 全部通过。
3. 测试覆盖 RBAC、CRUD、公开过滤、图片格式/大小/数量、存储失败、Excel dry-run/原子回滚/round-trip。
4. 本地 Compose 验证 PostgreSQL、MinIO、后端和前端健康。
5. 浏览器验证管理员商品管理、图片显示和 Excel 流程。
6. 远端部署前必须提供备份、迁移、健康检查和回滚步骤，并由独立 review 任务执行。
